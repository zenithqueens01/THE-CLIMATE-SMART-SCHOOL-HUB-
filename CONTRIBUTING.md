# Contributing to Climate Smart School Hub (CSSH)

Thank you for your interest in contributing to the Climate Smart School Hub!
We welcome contributions from developers, educators, climate advocates,
designers, and researchers worldwide.

---

## 🌍 Who Can Contribute?
- Developers (frontend, backend, mobile)
- UI/UX Designers
- Climate researchers and educators
- Technical writers and documentation experts
- Community advocates and NGO partners

---

## 🚀 How to Contribute

### Step 1: Fork the Repository
Click the "Fork" button at the top right of this page to copy 
the repo to your GitHub account.

### Step 2: Clone Your Fork
```bash
git clone https://github.com/YOUR_USERNAME/THE-CLIMATE-SMART-SCHOOL-HUB.git
cd THE-CLIMATE-SMART-SCHOOL-HUB
```

### Step 3: Create a New Branch
```bash
git checkout -b feature/your-feature-name
```
Examples:
- `feature/early-warning-ui`
- `fix/health-dashboard-bug`
- `docs/update-readme`

### Step 4: Make Your Changes
- Write clean, readable code
- Add comments where necessary
- Test your changes before submitting

### Step 5: Commit Your Changes
```bash
git add .
git commit -m "brief description of your change"
```

### Step 6: Push to Your Fork
```bash
git push origin feature/your-feature-name
```

### Step 7: Open a Pull Request
- Go to the original repo on GitHub
- Click "Compare & Pull Request"
- Describe your changes clearly
- Submit for review

---

## 📋 Contribution Guidelines

### Code Standards
- Use meaningful variable and function names
- Keep functions small and focused
- Comment complex logic
- Follow existing code style in the project

### Commit Message Format
- feat: add climate alert notification system
- fix: resolve mobile layout issue on dashboard
- docs: update installation guide
- style: improve button spacing on landing page
### Pull Request Standards
- One feature or fix per pull request
- Include screenshots for UI changes
- Reference any related issues using #issue-number
- Ensure your branch is up to date with main before submitting

---

## 🐛 Reporting Bugs
Open a GitHub Issue with:
- Clear title describing the bug
- Steps to reproduce it
- Expected vs actual behavior
- Screenshots if applicable
- Your device and browser details

---

## 💡 Suggesting New Features
Open a GitHub Issue with:
- Clear description of the feature
- Why it aligns with CSSH's mission
- How it would benefit schools or communities
- Any mockups or references if available
